# 学校モビー診断 解説記事用まとめ（実装ベース）

対象ソース: `16school/index.html`  
想定読者: 「学校モビー診断」の仕組みを記事で紹介したい人

---

## 1. 診断の全体像

- 質問数は **40問**（4軸 × 10問）。
- 回答形式は **7件法**（1〜7）。
- 診断結果は **4文字のひらがなコード**（例: `がやみそ`）。
- コードは4軸それぞれの左右どちらに寄ったかで決まる。
- 性別でデータセットが分岐:
  - 男子版: `AXES` / `QUESTIONS` / `CHARACTERS`
  - 女子版: `AXES_FEMALE` / `QUESTIONS_FEMALE` / `CHARACTERS_FEMALE`
- 質問順は毎回シャッフルされる（`questionOrder`）。
- 1ページ5問（`PAGE_SIZE = 5`）で進行し、全8ページ。

## 5. 結果データの構造（1タイプあたり）

各キャラクターには次の文言セットがある:

- `name`: キャラ名
- `summary`: 一言要約
- `catch`: キャッチコピー
- `hook`: 共感シーン説明
- `downside`: しんどくなる時の傾向
- `perception`: 周囲からの見え方
- `altCheck`: 隣タイプへの誘導文
- `midDesc`: 中間判定が多い時に使う説明文
- `strength`: 長所
- `weakness`: 注意点
- `love`: 恋愛傾向

表示時ロジック:

- `midCount <= 1` なら `hook` を主説明として表示
- `midCount > 1` なら `midDesc` を優先表示

## 6. 診断アルゴリズム（実装準拠）

### 6-1. 回答スケールと前処理

- 回答は `1..7`（中央は `4`）。
- `reverse:true` の質問は `ansAdj = 8 - ansRaw` に変換。
- これで全質問を「左寄りがプラス、右寄りがマイナス」に統一。

### 6-2. 重みづけと非線形変換

定数:

- `GAMMA = 1.35`
- `EXTREME_BONUS = 0.35`
- `COHERENCE_SHRINK = 0.20`

手順:

1. 端回答（1 or 7）は重み `1.35`、それ以外は `1.0`。
2. `centered = MID - ansAdj`（`MID=4`）
3. `t = centered / 3`（-1..+1）
4. `shaped = sign(t) * |t|^GAMMA`
5. `contrib = weight * shaped`
6. 軸ごとに `raw += contrib`, `absRaw += |contrib|`, `maxRaw += weight`

### 6-3. 軸スコアの確定

1. `coherence = |raw| / absRaw`（相殺が少ないほど1に近い）
2. `adjusted = raw * (1 - COHERENCE_SHRINK * (1 - coherence))`
3. `norm = clamp(adjusted / maxRaw, -1, 1)`
4. `hard = norm > 0 ? "L" : "R"`  
   - `norm == 0` は `"R"` 側になる実装

派生値:

- `pos = 50 - norm * 50`（表示用 0..100）
- `leftPct = round((norm + 1) * 50)`（左寄り%）
- `decisiveness = |norm|`

### 6-4. タイプコード生成

`hard(A,B,C,D)` に応じて各軸の `leftCode/rightCode` を連結して4文字コード化。  
例（男子）: `が + や + み + そ = がやみそ`

### 6-5. 確信度と隣接タイプ

軸ごとの閾値:

- `decisiveness >= 0.50` → `strong`（はっきり）
- `0.25 <= decisiveness < 0.50` → `moderate`（やや）
- `< 0.25` → `mid`（場面による）

中間軸（`mid`）は反対側に反転したコードも計算し、`adjacentTypes` として候補表示。

### 6-6. 全体確信度

- 4軸平均 `totalDecisiveness` を使用
- `>= 0.50` : `strong`
- `>= 0.30` : `moderate`
- `< 0.30` : `balanced`

### 6-7. 直線回答検知

- 同じ選択肢に偏る回答を検知する処理あり。
- 実装上は `1〜4` の頻度だけを最大比較しており、`5〜7` 側の偏りは同じ精度で拾えない点に注意。

## 7. 記事で押さえると伝わるポイント

1. 「性格の善悪判定」ではなく「学校生活での振る舞い傾向の可視化」。
2. 40問を4軸に分け、コード化して16タイプへ落とす設計が明快。
3. 中間判定（場面による）を明示し、単純断定を避ける実装になっている。
4. 結果文言は「共感シーン」「しんどい時」「周囲の見え方」まで揃っていて、読み物として成立しやすい。
