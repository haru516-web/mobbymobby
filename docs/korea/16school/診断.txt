学校モビー診断（16school）診断全体像
======================================

1. フォルダ構成（主要）
- index.html: 画面・デザイン・ロジック・診断データが全て入った単一SPA。
- api/lead.js: 診断登録（リード）送信API。Resendでメール送信。
- api/share.js: 共有・紹介イベント送信API。Resendでメール送信。
- cha/: キャラクター画像（男子版/女子版）一式。
- logo.jpg / favicon/fav.jpg: OG/ファビコン用画像。
- sample.html: 旧版/試作のHTML（index.htmlと近い構成）。
- package.json: Resend, @vercel/blob など依存。
- vercel.json: SPAリライト（/api以外はindex.html）。
- robots.txt / sitemap.xml: 検索向け。

2. 診断フロー（画面遷移）
- intro → genderSelect → quiz → gate → result → characters
- intro: 診断開始/再開。
- genderSelect: 男子版/女子版を選択。
- quiz: 40問を5問ずつ、全8ページ。未回答があると次へ進めない。
- gate: 結果閲覧前に登録（ニックネーム/年齢/メール/同意）。
- result: 診断結果の可視化・詳細解説・共有。
- characters: 16タイプの一覧と詳細モーダル。

3. 状態管理と保存
- localStorageキー:
  - school_char_diag_v1: 診断状態（回答・性別・ページなど）。
  - mobby_school_diag_pending_v1: Firebase連携失敗時の保留データ。
- state主項目:
  - step, page, answers, questionOrder, gender
  - profile(name/age/email), interested
  - userId, referrerId, shareCount
- リセットで全状態を破棄。

4. 診断データ（4軸×10問×2性別）
- 回答尺度: 7段階（1=そう思う … 7=そう思わない）
- 各軸10問（左肯定5問 + 右肯定5問）。右肯定は reverse:true。
- ページ構成: PAGE_SIZE=5 → 全8ページ。

軸の定義（男子版）
- A: キャラタイプ（陽キャラ/陰キャラ）→ が / な
- B: 雰囲気（やんちゃ/真面目）→ や / お
- C: 行動スタイル（仲間重視/個人行動重視）→ み / こ
- D: 評価基準（外=他者評価/内=自己基準）→ そ / う

軸の定義（女子版）
- A: キャラタイプ（陽キャラ/陰キャラ）→ が / な
- B: 雰囲気（清楚/ギャル）→ お / や
- C: 関係性（友達優先/恋愛優先）→ み / こ
- D: 評価基準（評価重視/自分基準）→ そ / う

5. スコアリング/判定ロジック（computeResult）
- 7段階回答を非線形変換して -1..+1 に正規化。
  - 中央(4)からの距離を強調: GAMMA=1.35
  - 端(1/7)は重み+35%（EXTREME_BONUS=0.35）
  - reverse:true は 8-ans で反転
- 軸内で左右が相殺される場合は中立寄せ（COHERENCE_SHRINK=0.20）。
- 4軸のhard判定（L/R）から4文字コードを生成。
- 左右比率を0..100%で出し、結果バーに反映。
- 直線回答検知: 1〜4の同一回答が80%以上で注意文を表示。
- 確信度（decisiveness）
  - 0.50以上: はっきり / 0.25以上: やや / それ未満: 場面による
- 中間軸は隣接タイプとして候補提示。
- 全体確信度: 4軸平均で strong/moderate/balanced を判定。

7. 結果画面の構成
- キャラ名/キャッチ/画像/TYPEコード。
- 4軸の比率バーと詳細（質問ごとの寄与・回答ラベル）。
- 中間軸がある場合は「場面によっては…」で隣接タイプ提示。
- 共感フック/しんどい時/周りからの見え方/長所/注意点。
- 4つの生活アドバイス（活躍場面、スタイル、人間関係、モチベーション）。

8. 登録（gate）とデータ送信
- 必須入力: ニックネーム/年齢/メール/同意チェック。
- 送信内容: 名前・年齢・メール・診断コード・各軸比率・回答・性別・共有情報。
- ローカル環境（localhost/127.0.0.1/file）は登録スキップ。

9. 共有・紹介・ユーザーID
- 8文字IDを生成し、?ref= で紹介トラッキング。
- share/line/x/copy の各イベントは /api/share に送信。
- 紹介訪問(referral_visit)・紹介完了(referral_complete)を記録。

10. Firebase連携
- firebase-app-compat / auth / firestore を使用。
- メール一致時に profiles コレクションへ schoolDiagnosis を保存。
- 失敗時は pending として localStorage 保留。

11. 興味ボタンと外部API
- 「このモビーが欲しい」ボタンで submitInterest。
- /api/lead に再送 + /api/interest に専用ページ送信を依頼。
- 注意: 本フォルダには api/interest.js が存在しないため、外部/別リポジトリ実装の可能性。

12. 画像とコードの対応
- CODE_TO_IMAGE / CODE_TO_IMAGE_FEMALE がコード→画像名を解決。
- 画像は cha/ 配下（例: ortp.jpg, ortp2.jpg など）。

13. デプロイ設定
- vercel.json で /api を除く全ルートを index.html にリライト。
- SPAとして動作し、APIは /api 以下のサーバレス関数で処理。
